# 法令データ自動管理システム (Law Data Monitor)

## 概要
本システムは、[e-Gov法令API](https://laws.e-gov.go.jp/api/2/swagger-ui)を利用して、指定した法令の最新データを自動取得し、人間およびLLM（生成AI）が読みやすいMarkdown形式に変換して履歴管理するシステムです。

法令改正の有無を定期的に自動監視し、変更があった場合は自動的に新旧対照表を生成・保存します。

## 主な機能
* **自動データ取得:** e-Gov APIから最新の法令データ（JSON形式）を取得。
* **Markdown変換:** 複雑な法令構造（編・章・節・条・項・号）を、マークダウンの見出しとリストを用いて構造化し保存。
* **自動更新・差分検知:** 月次で更新チェックを行い、改正があった場合のみファイルを更新。
* **新旧対照表の生成:** 更新があった際、変更前と変更後の条文テキストを比較し、変更箇所のみを抽出したMarkdownテーブル形式の対照表を自動生成。

## 🤖 LLM（生成AI）からの参照について
本リポジトリ内のデータは、RAG（検索拡張生成）システムなどの生成AIが外部知識として参照することを前提に構造化されています。

* **最新の法令データ:** `laws/` ディレクトリ内に `[法令名].md` の形式で格納されています。常に最新の施行データが維持されます。
* **改正履歴（差分）:** `changes/` ディレクトリ内に年月単位で格納されています。法改正の変遷や、旧規定・新規定の比較が必要な場合はこちらを参照してください。

## ディレクトリ構成

\`\`\`text
.
├── .github/workflows/
│   └── monthly_update.yml   # 月次自動更新のGitHub Actions定義
├── config/

└── src/                     # 処理用Pythonスクリプト
    ├── main.py              # メイン実行スクリプト
    ├── api_client.py        # API通信モジュール
    ├── converter.py         # JSON → Markdown変換モジュール
    └── diff_generator.py    # 差分比較・テーブル生成モジュール
\`\`\`

## 使い方・設定方法

### 1. 管理対象法令の追加・削除
取得対象の法令を変更するには、`config/target_laws.txt` を編集します。
e-Govの「法令番号（Law ID）」を1行に1つずつ記載してください。

**(例) config/target_laws.txt**
\`\`\`text
# 民法 (明治二十九年法律第八十九号)
129AC0000000089
# 会社法 (平成十七年法律第八十六号)
417AC0000000086
\`\`\`
※行頭が `#` の行はコメントとして無視されます。

### 2. ローカルでの実行方法
手元の環境で動作確認を行う場合は、以下のコマンドを実行してください。

**必須要件:** Python 3.9以上
\`\`\`bash
# 必要なライブラリのインストール
pip install requests

# スクリプトの実行
python src/main.py
\`\`\`

### 3. GitHub Actionsによる自動更新
本リポジトリは GitHub Actions と連携して自動運用されます。
* **実行タイミング:** 毎月1日（JST 09:00 / UTC 00:00）
* **処理内容:** `src/main.py` を実行し、ファイルに変更（差分）が生じた場合のみ、自動的にコミット＆プッシュを行います。
* **手動トリガー:** GitHubリポジトリの `Actions` タブ > `Monthly Laws Update` > `Run workflow` から、いつでも即時実行が可能です。

## 免責事項
* 本システムで取得・生成されるMarkdownデータは、e-Gov法令APIが提供する情報を機械的に変換したものです。データの正確性や完全性を保証するものではありません。
* 実務における法的な判断や確認には、必ず官報またはe-Govの公式Webサイト等で原本をご参照ください。
